MN/BOL
    Escenario F

    Rumores y Panico
    en la Red de Distribucion

    Se modela como una pequena perturbacion en la demanda puede amplificarse dentro de una red de abastecimiento. El analisis compara demanda normal contra demanda por rumores usando sistemas lineales, numero de condicion y sensibilidad.

    Contexto del Problema

    MODELO
    Situacion Bolivia: cuando circula el rumor de escasez, las familias compran mas combustible o alimentos de lo habitual. Aunque el cambio inicial parezca pequeno, una red mal condicionada puede producir asignaciones inestables entre zonas.
    A x = b
    A: restricciones entre zonas y rutas
    x: asignacion calculada por zona
    b: demanda observada normal o perturbada por rumor

    Datos de Simulacion

    FORMULARIO
    El factor de acoplamiento representa cuanto depende una zona de las demas. Mientras mas cercano a 1, el sistema queda peor condicionado y la red se vuelve mas sensible a rumores.

    Preguntas que Responde el Modelado

    RESPUESTAS

    1. Que provoca el rumor?

    Aumenta el vector b de demanda. Si la matriz A esta mal condicionada, la asignacion x cambia mas que la perturbacion original.

    2. Como se mide el riesgo?

    Con cond(A) y con el error relativo. Un valor alto indica que pequenos rumores generan decisiones numericas inestables.

    3. Que zona se prioriza?

    La zona con mayor incremento relativo debe recibir control de filas, informacion publica y reposicion planificada para reducir compras impulsivas.

    4. Cuando la solucion deja de ser confiable?

    Cuando cond(A) es alto y el cambio relativo en x supera claramente al cambio en b. En ese caso se recomienda redisenar restricciones o usar datos mas estables.

    Fundamento Numerico

    TEORIA
    cond(A) = ||A|| * ||A^-1||

    El numero de condicion estima la sensibilidad del sistema. Si cond(A) es grande, una variacion pequena en la demanda puede producir una variacion grande en la asignacion calculada.

    error relativo en x <= cond(A) * error relativo en b

    El rumor entra como perturbacion del vector de demanda. La desigualdad permite decidir si el resultado es confiable o si debe tratarse como escenario de alto riesgo.

    x = A^-1 b

    Resolver el sistema permite comparar la red antes y despues del rumor, manteniendo las mismas rutas y cambiando solo la demanda observada.
