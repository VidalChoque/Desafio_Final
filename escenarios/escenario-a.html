MN/BOL
    ⬡ Escenario A

    Optimización del Abastecimiento
    y Red de Transporte

    Se simula la distribución de carburantes y alimentos desde plantas de acopio hacia zonas de la ciudad, considerando restricciones de bloqueos y capacidad de rutas. Se utiliza un sistema de ecuaciones lineales resuelto con múltiples métodos iterativos.

    Contexto del Problema

    MODELO
    Situación Bolivia: Con los bloqueos en carreteras y la escasez de carburantes, las plantas distribuidoras deben optimizar cuánto enviar a cada zona (Norte, Centro, Sur) desde sus depósitos. El modelo matemático permite calcular la distribución óptima y analizar qué ocurre cuando una ruta queda inhabilitada.

    El sistema de ecuaciones representa restricciones de oferta, demanda y capacidad:

    Planta 1 → [x₁, x₂, x₃] → [Zona Norte, Zona Centro, Zona Sur]
    Planta 2 → [x₄, x₅, x₆]
    Planta 3 → [x₇, x₈, x₉]

    Ax = b → donde A es la matriz de coeficientes, x el vector solución

    Ingreso de Datos del Sistema

    FORMULARIO
    Sistema por defecto: Distribución de combustible en litros/día. Edita los valores de la matriz A y el vector b.

    Preguntas que Responde el Modelado

    RESPUESTAS

    1. Cuanto debe enviarse a cada zona?

    La solucion del sistema Ax=b entrega la asignacion por zona. En la simulacion, cada valor x representa litros por dia que deben dirigirse a Norte, Centro, Sur u otras zonas activadas.

    2. Que pasa si una ruta queda bloqueada?

    Al modificar la matriz A o el vector b, el sistema redistribuye la carga. Una zona puede recibir menos abastecimiento y otra absorber mayor demanda, mostrando el impacto directo del bloqueo.

    3. Que metodo conviene usar?

    LU da una solucion directa; Jacobi, Gauss-Seidel, SOR y Gradiente Conjugado permiten comparar convergencia. Si la matriz es diagonalmente dominante, los metodos iterativos son confiables.

    4. Como se valida la respuesta?

    Se revisa el residuo maximo |b-Ax|, el numero de iteraciones y la dominancia diagonal. Un residuo cercano a cero indica que la distribucion calculada satisface las restricciones.

    Algoritmos Implementados

    TEORÍA
    x_i^(k+1) = (b_i - Σ a_ij · x_j^(k)) / a_ii (j ≠ i)

    El método de Jacobi actualiza todas las variables simultáneamente usando los valores de la iteración anterior. Requiere que la matriz sea diagonalmente dominante para converger. Ideal para sistemas donde cada zona se abastece independientemente.

    x_i^(k+1) = (b_i - Σ a_ij·x_j^(k+1) [j<i] - Σ a_ij·x_j^(k) [j>i]) / a_ii

    Gauss-Seidel usa valores actualizados inmediatamente. Converge más rápido que Jacobi para sistemas bien condicionados. Modela mejor la realidad: al redistribuir a zona Norte, ese dato ya se usa para calcular Zona Centro.

    x_i^(k+1) = (1-ω)·x_i^(k) + ω·x_GS^(k+1)

    SOR (Successive Over-Relaxation) acelera Gauss-Seidel con un factor ω. Con ω=1 es Gauss-Seidel. Para ω∈(1,2) hay sobre-relajación (más rápido). Para sistemas con bloqueos parciales, SOR permite ajustar la velocidad de convergencia.

    A = L·U → L·y = b → U·x = y

    La factorización LU descompone A en triangular inferior L y superior U. Exacto para sistemas de tamaño moderado. Útil cuando se resuelven múltiples escenarios con la misma matriz A (distintos vectores b: demanda normal vs emergencia).

    x^(k+1) = x^(k) + α_k·d^(k)
    α_k = (r^T·r) / (d^T·A·d)

    El Gradiente Conjugado minimiza el residuo en el espacio de Krylov. Eficiente para sistemas grandes y simétricos definidos positivos. Útil en redes de distribución extensas con muchas rutas.
