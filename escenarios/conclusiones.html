    Conclusiones Generales
    del Modelado Numerico

    Esta pagina resume lo aprendido al aplicar metodos numericos a los siete escenarios: abastecimiento, reservas, precios, gasto familiar, umbrales, rumores y dinamica social.

    Que se Aprendio

    SINTESIS

    Modelar ayuda a decidir

    Los metodos numericos convierten datos dispersos en respuestas medibles: cuanto distribuir, cuando se agotan reservas, cuanto sube un precio o cuando aparece un umbral critico.

    El contexto cambia el resultado

    Bloqueos, rumores, panico y mediacion no son detalles externos: modifican matrices, ecuaciones, parametros y curvas. Por eso el modelo debe permitir simular escenarios.

    La precision importa

    Un paso h grande, datos incompletos o una matriz mal condicionada pueden cambiar la conclusion. La validacion con residuo, error e iteraciones es parte del analisis.

    El resultado debe interpretarse

    Un numero aislado no basta. Cada escenario traduce el calculo a una decision: priorizar zonas, reponer reservas, anticipar gasto o reducir tension social.

    Metodo Mas Util

    COMPARACION
    Conclusion principal: no existe un unico metodo mejor para todo el proyecto. El metodo mas util depende del tipo de pregunta: sistemas lineales para distribucion, EDO para evolucion temporal, interpolacion para datos faltantes, integracion para gasto acumulado y raices para umbrales.
    EscenarioMetodo mas utilPor que fue util
    A: AbastecimientoLU / Gauss-SeidelResuelven la asignacion de recursos y permiten verificar restricciones de la red.
    B: ReservasRK4Predice con mejor precision el vaciado de reservas ante demanda creciente o panico.
    C: PreciosSplines cubicosReconstruyen curvas suaves y evitan oscilaciones fuertes entre datos de mercado.
    D: Gasto familiarSimpson 1/3Calcula el gasto acumulado con alta precision para curvas de precios progresivas.
    E: UmbralesBiseccion / SecanteEncuentran puntos criticos de forma clara; biseccion es robusta y secante evita derivadas.
    F: RumoresNumero de condicionMide la sensibilidad de la red frente a perturbaciones pequeñas en la demanda.
    G: Descontento socialRK4Representa mejor la evolucion no lineal de neutrales, manifestantes y mediadores.

    Respuesta general

    El metodo mas completo para evolucion temporal fue RK4, porque aparece en reservas y dinamica social con buena precision. Para decisiones de abastecimiento, los sistemas lineales fueron los mas practicos, porque entregan una distribucion directa de recursos.

    Limitaciones del Modelo

    CRITICA

    Datos simplificados

    Los valores usados son academicos o ajustables. No reemplazan datos oficiales de inventarios, precios, rutas, consumo real o comportamiento social.

    Supuestos fuertes

    Varios escenarios suponen crecimiento lineal, exponencial o relaciones constantes. En una crisis real los parametros cambian por decisiones politicas, clima, transporte y mercado informal.

    Interaccion parcial

    Los escenarios se analizan por separado. En la realidad, precios, reservas, rumores y conflicto social se retroalimentan al mismo tiempo.

    Validacion limitada

    El proyecto valida numericamente con errores, residuos y graficas, pero no calibra los modelos con una base historica completa.

    Alcance academico: los resultados sirven para comprender metodos numericos y explorar decisiones posibles, no para emitir predicciones definitivas sobre una situacion social real.

    Mejoras Futuras

    PROPUESTAS

    Datos reales conectados

    Importar precios, reservas, consumo y rutas desde archivos CSV o una API para actualizar los escenarios sin editar el codigo.

    Modelo integrado

    Unir reservas, precios, rumores y descontento en un solo sistema donde cada resultado afecte a los demas.

    Analisis de sensibilidad

    Variar automaticamente parametros como demanda, ingreso, tasa de reposicion y mediacion para identificar cuales influyen mas.

    Alertas visuales

    Agregar semaforos, mapas de riesgo y recomendaciones automaticas cuando se cruce un umbral critico.

    Comparacion historica

    Calibrar los modelos con datos de periodos anteriores para medir error real y ajustar parametros.

    Exportacion de resultados

    Permitir descargar tablas y graficas en PDF o Excel para presentar el analisis completo del grupo.

    Conclusion Final

    CIERRE

    El proyecto demuestra que los metodos numericos permiten transformar una situacion compleja en modelos calculables, comparables y visuales. Aunque cada escenario tiene limites, el conjunto muestra una idea central: con datos, supuestos claros y validacion numerica, es posible anticipar riesgos, justificar decisiones y entender mejor la relacion entre abastecimiento, precios y comportamiento social.
